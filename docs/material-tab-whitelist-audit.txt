SoE Companion Material Tab Whitelist Audit

Purpose:
This file separates two ideas that the app currently mixes together:
1. Trackable materials: items SoE Companion can count, notify, or use for achievements.
2. Materials-tab storage items: items the in-game Materials tab will actually accept.

The stash sorter should only auto-route items to the Materials tab when they are in the confirmed Materials-tab storage whitelist.
Everything else should follow explicit user rules or stay untouched.

Current automatic Materials-tab logic in native/drop-hook-source/ijl11_grail.cpp:
- Any rune code r01-r33, including stacked variants like r01s.
- Any listed gem code, including stacked variants.
- Any listed known material code, including stacked variants, excluding known non-material-tab families.
- Any generic flp* stack wrapper whose display name contains one of the allowed material display strings.
- Boss essence codes only: tes, ceh, bet, fed.

High-risk broad display-name matches currently used by the hook:
- rune
- twisted essence
- charged essence
- burning essence
- festering essence

These broad display-name checks should not be trusted as the final Materials-tab whitelist.
The hook now blocks terrors, glyphs, chisels, Token of Absolution, Hatred Orbs, and Ascendancy/Soulstone items from the automatic Materials Tab path.

Next implementation direction:
1. Create a shared `MATERIAL_TAB_ITEMS` whitelist with exact item names and exact item codes.
2. Use that whitelist for the Stash Sorter Automatic Materials Tab toggle.
3. Keep `MATERIAL_NAMES` for sounds, achievements, and overlays.
4. Remove broad automatic checks from `IsSorterAutomaticMaterialTabItem`; `es*`, `hor*`, and broad `as*` have been removed or blocked.
5. Keep user-created category rules separate from automatic material affinity.
